# 🎬 Festival Scout IA

## Qu'est-ce que c'est ?

Un projet pour **automatiser la recherche et la préparation de candidatures** pour les festivals de cinéma.

### Le pipeline :

```
🎬 MON FILM
    ↓
🔍 Recherche festivals compatibles
    ↓
📋 Extraction des infos festival
    ↓
🧮 Analyse compatibilité
    ↓
⭐ Priorisation stratégique
    ↓
📝 Préparation candidature béton
    ↓
📧 Envoi emails festival
    ↓
🔔 Suivi & relances
    ↓
📊 Dashboard de suivi
```

## 🚀 Démarrer

### Étape 1 : Lire la doc
1. Lire [GITHUB_WORKFLOW_FESTIVAL_SCOUT_IA.md](docs/GITHUB_WORKFLOW_FESTIVAL_SCOUT_IA.md)
2. Comprendre les 9 colonnes Kanban
3. Comprendre la règle : "To refine → Ready"

### Étape 2 : Créer les labels
1. Aller sur : `https://github.com/nassimus13/Scout-Festival/labels`
2. Suivre [LABELS_FESTIVAL_SCOUT_IA.md](docs/LABELS_FESTIVAL_SCOUT_IA.md)
3. Créer les 23 labels

### Étape 3 : Créer le Project Kanban
1. Aller sur : `https://github.com/nassimus13/Scout-Festival/projects`
2. Créer un nouveau "Kanban" project
3. Créer les 9 colonnes (voir workflow)

### Étape 4 : Créer les 5 tickets initiaux
1. Suivre [INITIAL_ISSUES_FESTIVAL_SCOUT_IA.md](docs/INITIAL_ISSUES_FESTIVAL_SCOUT_IA.md)
2. Créer les 5 issues
3. Les ajouter au Project en colonne "To refine"

### Étape 5 : Clarifier les tickets
1. Lire chaque ticket
2. Poser des questions si besoin
3. Passer les tickets clairs à "Ready"

---

## 🎯 Principes clés

✅ **Un ticket clair > un ticket rapide**  
On prend le temps de bien clarifier avant de commencer.

✅ **Une tâche = une issue**  
Pas d'issues géantes avec 10 tâches dedans.

✅ **Labels + Kanban = traçabilité**  
On sait toujours où on est et pourquoi.

✅ **Test avant livrable**  
Pas de ticket en "Livrable" sans validation.
